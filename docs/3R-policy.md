# 3R Policy

This repository interprets the 3R contribution claimed in the paper as follows:

- **Repeat**: rerun the original benchmark implementations and regenerate raw CSV files.
- **Reproduce**: regenerate the published analytical artefacts from preserved raw or processed data.
- **Replicate**: execute the same protocol on another ARM Morello / CheriBSD installation.

## Section-level status

- **Section 4.1 Memory Performance** — repeatable, reproducible, replicable.
- **Section 4.2 Communication Performance** — repeatable and replicable from source; partially reproducible because the published analytical summary was assembled manually and is archived rather than regenerated by an authoritative script.
- **Section 4.3 Cryptographic Library Performance** — repeatable, reproducible, replicable.
