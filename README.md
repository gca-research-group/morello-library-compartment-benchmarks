# Morello Library-Based Compartment Benchmarks

This repository organises the artefacts associated with the paper **“Software and Hardware Technologies for Security in Computing Systems: Performance Evaluation of a Library--Based Compartment Approach on ARM Morello Board with CheriBSD”** into a structure aligned with the **3R** principles declared in the article: **Repeat, Reproduce, Replicate**.

The study evaluates three experimental axes on ARM Morello Board with CheriBSD: **memory operations**, **Unix pipe communication**, and **cryptographic library workloads**. The three execution modes used throughout the study are:

- **outside compartment** (baseline)
- **purecap ABI**
- **purecap-benchmark ABI**

## 3R status by section

| Section | Experiment | Repeat | Reproduce | Replicate | Status |
|---|---|---:|---:|---:|---|
| 4.1 | Memory Performance | Yes | Yes | Yes | Fully scripted |
| 4.2 | Communication Performance | Yes | Partially | Yes | Raw benchmark preserved; published analysis archived as manual artefact |
| 4.3 | Cryptographic Library Performance | Yes | Yes | Yes | Fully scripted |

### Interpretation of the 3R categories used here

- **Repeat**: rerun the original benchmark code and regenerate raw CSV files.
- **Reproduce**: regenerate published tables/figures from preserved raw or processed data.
- **Replicate**: execute the same protocol on another Morello/CheriBSD setup.

## Repository map

- `experiments/01_memory/` — source code, binaries, and raw CSV files for Section 4.1
- `experiments/02_communication/` — source code, binaries, and raw CSV files for Section 4.2
- `experiments/03_crypto/` — source code, headers, Makefile, binary, and raw CSV files for Section 4.3
- `analyses/01_memory/` — scripted analysis and outputs for Section 4.1
- `analyses/02_communication/` — archived manual analysis artefacts for Section 4.2
- `analyses/03_crypto/` — scripted analysis and outputs for Section 4.3
- `docs/` — environment, provenance, 3R policy, and result mapping
- `scripts/` — convenience wrappers and repository verification helpers

## Important integrity note

All original experiment, analysis, CSV, SVG, XLSX, PDF, source-code, and binary artefacts were **reorganised without changing file contents**. New documentation files were added around them, but the preserved artefacts themselves were copied byte-for-byte into the current structure. See `MANIFEST.sha256` for the preservation map and checksums.

## Environment summary

The paper evaluates the benchmarks on **ARM Morello Board** running **CheriBSD 24.5 / 24.05** with `clang-morello`, `proccontrol`, and the `purecap` / `purecap-benchmark` ABIs. See `docs/environment.md` for details.

## Exact commands to compile and execute each experiment

All commands below assume execution on **ARM Morello Board with CheriBSD**, starting from the repository root.

---

### Experiment 4.1 — Memory Performance

#### Outside compartment (baseline)

```bash
cd experiments/01_memory
mkdir -p rerun/outside
clang-morello -g -o bin/memory-out-experiment src/memory-out-experiment.c -lm
(
  cd rerun/outside
  ../../bin/memory-out-experiment
)
```

Expected output file generated by the preserved source:

- `experiments/01_memory/rerun/outside/memory-out-experiment-results.csv`

#### purecap ABI

```bash
cd experiments/01_memory
mkdir -p rerun/purecap
clang-morello -march=morello -mabi=purecap -g -o bin/memory-in-experiment-purecap src/memory-in-experiment-purecap.c -lm
(
  cd rerun/purecap
  proccontrol -m cheric18n -s enable ../../bin/memory-in-experiment-purecap
)
```

Expected output file generated by the preserved source:

- `experiments/01_memory/rerun/purecap/memory-in-experiment-results.csv`

#### purecap-benchmark ABI

```bash
cd experiments/01_memory
mkdir -p rerun/benchmark
clang-morello -march=morello -mabi=purecap-benchmark -g -o bin/memory-in-experiment-purecap-benchmark src/memory-in-experiment-purecap-benchmark.c -lm
(
  cd rerun/benchmark
  proccontrol -m cheric18n -s enable ../../bin/memory-in-experiment-purecap-benchmark
)
```

Expected output file generated by the preserved source:

- `experiments/01_memory/rerun/benchmark/memory-in-experiment-benchmarkABI-results.csv`

If you want to compare the rerun outputs against the archived raw files, compare them to:

- `experiments/01_memory/raw/memory-out-experiment-results.csv`
- `experiments/01_memory/raw/memory-in-experiment-purecap-results.csv`

### Experiment 4.3 — Cryptographic Library Performance

The preserved `Makefile` can still be used, but the commands below avoid overwriting the archived binary in `experiments/03_crypto/bin/` and make the three execution modes explicit.

#### Outside compartment (baseline)

```bash
cd experiments/03_crypto
mkdir -p rerun/build rerun/raw
clang-morello -O2 -Wall -Wextra -Wpedantic -std=c11 -Iinclude   src/crypto_bench.c src/crypto_workload.c -lcrypto   -o rerun/build/crypto_bench_outside
./rerun/build/crypto_bench_outside   --mode outside   --workload all   --repetitions 100   --warmup 1   --csv rerun/raw/crypto_outside.csv
```

#### purecap ABI

```bash
cd experiments/03_crypto
mkdir -p rerun/build rerun/raw
clang-morello -O2 -Wall -Wextra -Wpedantic -std=c11 -Iinclude   -march=morello -mabi=purecap   src/crypto_bench.c src/crypto_workload.c -lcrypto   -o rerun/build/crypto_bench_purecap
proccontrol -m cheric18n -s enable ./rerun/build/crypto_bench_purecap   --mode purecap   --workload all   --repetitions 100   --warmup 1   --csv rerun/raw/crypto_purecap.csv
```

#### purecap-benchmark ABI

```bash
cd experiments/03_crypto
mkdir -p rerun/build rerun/raw
clang-morello -O2 -Wall -Wextra -Wpedantic -std=c11 -Iinclude   -march=morello -mabi=purecap-benchmark   src/crypto_bench.c src/crypto_workload.c -lcrypto   -o rerun/build/crypto_bench_benchmark
proccontrol -m cheric18n -s enable ./rerun/build/crypto_bench_benchmark   --mode benchmark   --workload all   --repetitions 100   --warmup 1   --csv rerun/raw/crypto_benchmark.csv
```

Archived raw references:

- `experiments/03_crypto/raw/crypto_outside.csv`
- `experiments/03_crypto/raw/crypto_purecap.csv`
- `experiments/03_crypto/raw/crypto_benchmark.csv`

## Known limitation

The communication benchmark in Section 4.2 is preserved as:
- raw benchmark code and raw CSV outputs
- archived processed summaries (`experiments_summary.csv`, `experiments_summary_no_outliers.csv`, and related spreadsheet artefacts)
- manual-analysis documentation

It is therefore **repeatable** and **inspectable**, but not fully script-reproducible end-to-end from raw CSV to the exact published table with the materials currently preserved here.
